 Rules 

Build production-ready React Native \+ Expo code using TypeScript.

Rules:  
\- Do not use inline styles.  
\- Use Tamagui tokens only.  
\- Keep UI, business logic, API calls, and types separated.  
\- No hardcoded colors, spacing, or font sizes.  
\- Use reusable components.  
\- Use feature-based folders.  
\- Add loading, empty, error, and success states.  
\- Handle edge cases.  
\- Do not place Supabase calls directly inside UI components.  
\- Use services for backend calls.  
\- Use hooks for screen logic.  
\- Keep components small and readable.  
\- Avoid duplicate logic.
